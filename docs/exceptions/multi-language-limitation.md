# Exception: multi-language content and screen-reader pronunciation

## Summary

SWOT Builder allows users to enter subjects and receive AI-generated interview questions and suggestions in any natural language. Content entered by users, and content generated by the AI, is rendered inside the application without dynamic `lang` attribute management. Screen readers (VoiceOver, JAWS, NVDA) apply the page's declared language — `en-GB` — to all content, including content in other languages.

## WCAG criterion

WCAG 3.1.2 Language of Parts (Level AA): "The human language of each passage or phrase in the content can be programmatically determined except for proper names, technical terms, words of indeterminate language, and words or phrases that have become part of the vernacular of the immediately surrounding text."

## Impact

A French-speaking user who types a French subject and receives French questions and suggestions will hear those French words read aloud by a screen reader using English pronunciation rules. The words will be intelligible in writing but may be mispronounced in audio.

## Why this is accepted

SWOT Builder is a single-page browser application with no server-side rendering and no user accounts. The language of any given user session is not known at build time. Adding dynamic `lang` attribute management would require detecting the language of individual text nodes as they are inserted by the AI stream, which is a substantial engineering change and carries accuracy risk.

The impact is limited: the application's interface chrome (buttons, labels, navigation) is entirely in English. Only user-entered and AI-generated content would be affected, and only for users whose content is not in English.

## Acceptance

Accepted as a known limitation at the v1.0.0 release. Tracked as item L-04 in the project's accessibility remediation log.

## Resolution path

If SWOT Builder is ever internationalised, this exception should be revisited. Resolution would involve:

1. A language-detection library (for example `franc` or `langdetect`) applied to incoming AI stream chunks.
2. Wrapping detected non-English passages in a `<span lang="xx">` element with the detected language code.
3. Updating the exception record to `resolved` once automated tests confirm language tagging is present and correct.

Until then, this limitation is documented and accepted.
